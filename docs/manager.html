TEAM Engine 2.0 Web Application

TEAM Engine 2.0 comes with a web application to enable users to execute test suites online. The web application runs as a set of servlets in Apache Tomcat.

Building the Web Application

The web application is distributed in source format only. You must rebuild it with the CTL test suite scripts you want to deploy to create a complete webapp that can be deployed in Tomcat.

  1. Obtain the source code and binary dependencies. There are two ways to do this:

    1. Obtain the source from the subversion repository on SourceForge. Check out this URL:

      https://teamengine.svn.sourceforge.net/svnroot/teamengine/branches/team2
    2. Download the necessary packages from SourceForge download page. You will need all three distribution files, teamengine-bin.zip, teamengine-src.zip, and teamengine-manager-src.zip. Unpack them all into the same location.

  2. Add your custom CTL scripts. Unzip the script packages that you want to install inside the teamengine directory. This should create or add to a scripts directory, at the same level as the apps directory.

  3. If necessary, add custom java components. Unzip the component packages that you want to install inside the teamengine directory. This should create or add to a components directory, at the same level as the apps directory.

  4. Set user-specific properties. Create a build.properties file in the teamengine directory, using build.properties.example from the SVN repository as an example. You may set the following properties:

    Property

    Required

    Description

    version.stamp

    No

    Version number used in teamengine directory and .zip archives.

    usersdir

    No

    Users directory on the server to be used by the webapp.

    workdir

    Yes

    Working directory on the server to be used by the webapp.

    tomcat.dir

    Yes

    Location of a local tomcat installation (for locating dependency jar files necessary for building the webapp)

  5. Run Apache ant on the main build file. The following distribution files are generated.

    • zips/teamengine{version}-bin.zip
    • zips/teamengine{version}-src.zip
    • zips/teamengine-manager{version}-src.zip
    • zips/teamengine-manager{version}-bin.zip
    • zips/ccc.zip (An archive is created for each installed component)
    • zips/sss.zip (An archive is created for each installed script)

Installing the Web Application

  1. Install JDK 1.5.0 or greater.
  2. Install Tomcat 5.x or greater.
  3. Create a working directory, corresponding to the workdir setting used in the build.properties file.
  4. If you specified a usersdir setting used in the build.properties file, create the corresponding directory.
  5. Copy the UserFilesRealm.jar file from the setup directory into the Tomcat server/lib directory.
  6. Copy teamengine.war from the setup directory into the Tomcat webapps directory.
  7. Restart Tomcat.

The TEAM Engine Manager web application should now be running at http://localhost:8080/teamengine.
